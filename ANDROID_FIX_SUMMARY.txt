================================================================================
WHAT WAS BROKEN
================================================================================

Problem: VoiceStateMachine → AudioPipeline → getUserMedia() failing silently
Impact: Android users saw "❌ Failed to initialize" with no explanation
Root Cause: 
  1. Permission checks not performed (user previously denied → silent fail)
  2. Audio constraints too strict for Android hardware (echoCancellation + noiseSuppression)
  3. No error reporting to user
  4. No diagnostic information available

================================================================================
WHAT WAS FIXED
================================================================================

1. PERMISSION PRE-FLIGHT CHECK ✅
   - Uses Permissions API to check status before getUserMedia()
   - Shows clear error if permission denied
   - Enables fast failure instead of silent timeout

2. CONSTRAINT FALLBACK (4-LEVEL STRATEGY) ✅
   - Level 0: Full features (desktop)
   - Level 1: Reduced features (modern Android)
   - Level 2: Minimal features (mid-range Android)
   - Level 3: Any microphone (legacy Android)
   - Automatically tries less strict constraints if hardware doesn't support

3. ERROR-SPECIFIC REPORTING ✅
   - NotAllowedError → "Permission denied - see Settings"
   - NotFoundError → "No microphone found"
   - NotReadableError → "Microphone in use"
   - SecurityError → "HTTPS required"
   - Each error has actionable user message

4. PERMISSION FIX UI ✅
   - Yellow warning box appears if permission denied
   - Step-by-step fix instructions (Android & iOS specific)
   - Embedded diagnostic information
   - User doesn't have to search for help
